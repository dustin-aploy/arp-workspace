# Role Boundaries

A good ARP example does not describe a generic “helpful AI assistant.” It describes a bounded organizational role.

## What role boundaries should show

Each example should make it obvious:

- who owns the agent;
- what work is allowed;
- what work is prohibited;
- what approvals are required;
- when escalation must happen;
- what memory is allowed to persist; and
- what budget and performance limits govern the role.

## Why role boundaries matter

The examples in this directory are meant to teach that ARP is not primarily about model capability. It is about operational responsibility. A realistic example should therefore make the “no” boundaries, handoffs, audit expectations, and review cycles concrete.

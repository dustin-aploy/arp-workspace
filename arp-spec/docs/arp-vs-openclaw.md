# ARP vs. OpenClaw

ARP is protocol-oriented, not runtime-oriented. Where an agent system may emphasize execution, tooling, or workflow composition, ARP focuses on responsibility boundaries: identity, scope, authority, escalation, audit, memory, budget, and performance.

If an OpenClaw-based system wanted to interoperate with ARP, it would treat ARP as a source of policy structure and compliance metadata rather than as a replacement for its execution model.
